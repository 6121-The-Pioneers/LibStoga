void ls::AbstractOdom::compute ( )
virtual

Computes the new coordinations and angle of the robot since the program has started. Assumes that (0, 0) is where the robot starts at reboot. Also assumes that initial posture is 0 degrees (bearing)

Must give X, Y, and angle new values after this call is over.

Exceptions

◆ initialize()

virtual void ls::AbstractOdom::initialize ( std::initializer_list< uint8_t > ports)
pure virtual

Initialize this AbstractOdom with a list of pins. Depending on the specific subclass, the meaning of this and the specific order of the pins may be different.

Ex. The 3 Wheel Odom class can have an instance where the 1st one is center, 2nd is right, 3rd is left But for 2 Wheel Odom + IMU, it might have an instance where 1st one is IMU, 2nd is perpendicular, 3rd is parralel.

If items on ports have to be reversed, it can be given as a negative number

Parameters
portsthe ports that this Odom object uses

Implemented in ls::ImuOdom, ls::MCLTracker, and ls::ThreeWheelOdom.
